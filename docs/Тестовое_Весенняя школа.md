# Собери своего AI-агента

## Зачем это

Мы отбираем участников в летнюю школу, где вы будете решать открытые ИИ-задачи от реальных компаний. Этот мини-хак — способ для нас увидеть, как вы думаете и принимаете инженерные решения, а для вас — попробовать современные подходы к построению AI-агентов на практике.

Мы оцениваем не «идеальность», а:
- осмысленность решения  
- умение работать с инструментами  
- креативность и честную рефлексию  

## Что надо сделать

Собрать работающего AI-агента в одном из четырёх треков.

Результат — 4 артефакта:

1. Репозиторий на GitHub/GitLab  
   С README и инструкцией запуска (одна команда)

2. Скринкаст 60–90 секунд  
   Показать 1–2 сценария работы агента  
   Озвучка необязательна

3. Эссе (REFLECTION.md, 300+ слов)  
   Три блока:
   - Почему эта задача — что зацепило лично, для кого решаете  
   - Где агент сломался — 1–2 честных failure case  
   - Что сделали бы дальше при наличии времени  

4. Тег трека  
   Первая строка в README:  
   `Track: A / B / C / D` (или комбинация, например A+B)

## Жёсткие правила

- У агента минимум 1 инструмент (tool)  
  (2 инструмента — сильный плюс)

- Агент должен реально использовать инструмент  
  Просто вызвать LLM — не засчитывается

- LLM — любой доступный  
  (OpenAI, Claude, Gemini, YandexGPT, Groq, OpenRouter, локальные)

- Стек — любой  
  (LangGraph, LlamaIndex, smolagents, Agents SDK, n8n, Python и т.д.)

- Дедлайн — [указать дату и время]  
- Сдача — одна ссылка на репозиторий через [форма/бот]

## Треки

Выбираете один. Можно честно комбинировать два (например B+C)

## Трек A. Агент + API

Соберите агента, который использует внешний API для ответа на запросы

Что нужно сделать:
- агент принимает запрос пользователя  
- при необходимости вызывает API  
- обрабатывает результат  
- формирует ответ  

Примеры задач:
- «Найди последние новости по теме X»  
- «Покажи информацию о GitHub-репозитории»  
- «Какая погода в городе?»  

Минимум:
- 1 API как инструмент  
- агент сам решает, когда его вызывать  

## Трек B. Агент + классический ML

Соберите агента, который использует вашу ML-модель как инструмент

Что нужно сделать:
- обучить простую модель (sklearn / аналог)  
- сохранить модель  
- подключить её к агенту  
- использовать результат модели в ответе  

Примеры задач:
- классификация текста  
- определение тональности  
- простой прогноз  

Что обязательно:
- train.ipynb или скрипт обучения  
- метрика (accuracy / f1 и т.д.)  
- модель загружается в агенте  

## Трек C. Data-агент

Соберите агента, который работает с данными и отвечает на вопросы

Что нужно сделать:
- получить данные (API, парсинг или готовый файл)  
- привести к структуре (JSON/CSV)  
- использовать данные при ответах  

Примеры задач:
- сравнение товаров  
- новостной дайджест  
- подбор вариантов по параметрам  

Что обязательно:
- ссылки на данные лежат в репозитории  
- указаны источники и дата сбора  
- ответы опираются на данные  

## Трек D. Агент для оптимизации

Соберите агента, который решает задачу распределения или планирования

Что нужно сделать:
- принять входные условия  
- сформулировать задачу  
- найти решение  
- объяснить результат  

Примеры задач:
- распределение бюджета  
- составление расписания  
- построение маршрута  

Что обязательно:
- есть baseline (простое решение)  
- есть улучшенное решение  
- есть сравнение  

## FAQ

Можно взять open-source шаблон?  
Можно, но в README нужно указать, что использовали и что сделали сами

Что если агент иногда ломается?  
Это нормально. Опишите это в REFLECTION.md — это плюс

Обязательно ли 2 инструмента?  
Минимум 1. Два — усиливают решение

Нужен ли UI?  
Нет. CLI или ноутбук достаточно. При желании — можно добавить
